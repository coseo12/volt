각 엔트리 구조:
- **처리량** — 이번 배치에서 정제한 이슈/노트 수
- **관찰** — 흐름에서 눈에 띈 것(마찰·품질·일관성)
- **시그널** — 앞으로의 방향에 주는 함의

- **frontmatter 정규화 운영 반영**: 5차 배치 시그널("규약 명문화 여지")을 이번 refine 에 즉시 반영 — `type: report` + `report_type: decision` → `type: decision` 정규화를 refine 커밋 하나에 포함(별도 `fix` 커밋 없음). 운영 규약이 한 단계 굳어짐.

- **헤더 이름 규약 일탈 발생 → 파서 자동화 전제 흔들림**: 9차까지 0 이었던 "헤더 이름 규약 일탈" 이 최초 발생. 이슈 폼의 강제력이 우회되는 경로(수동 작성 + 잘못된 템플릿 선택) 가 실존함을 확인. 자동화 설계 시 "라벨 vs 본문 템플릿" 불일치 감지·자동 교정 또는 경고가 필요.
- **report_type 추론 규칙 명문화 필요**: `### 리포트 유형` 헤더 부재 시 본문 구조(일반화 요소 유무) 기반 fallback 이 이번 배치의 실질 기준. AGENTS.md 나 정제 가이드에 "헤더 부재 시 추론 우선순위(본문 → 제목 → 폴백: research)" 명문화 여지.
- **단일 에피소드 분할 캡처의 RAG 함의**: "1 에피소드 = 다수 관점 이슈" 가 표준 캡처 스타일로 정착하면, RAG 인덱스 설계 시 "에피소드 ↔ 이슈 ↔ 노트" 3단 인덱싱 또는 **에피소드 태그(source_repo + release 범위)** 2차 지표 가치 상승. 10차 배치가 첫 근거 사례.
- **관계 그래프 정합성 검증 유틸 필요성 상승**: 4노드 + 기존 노트 3개 참조 = 7노드 그래프가 한 배치에 형성. 수동 백필 4건 실수 없이 가능했지만, 다음에 더 큰 클러스터(8~10노드) 발생 시 경로 오류·누락 위험 급증. `related` 배열 정합성 검증(양방향 일치, 경로 유효성) 스크립트 후보.
- **교차타입 우세 고착**: 동타입 쌍 4 vs 교차타입 쌍 6. 9차에서 포착한 "타입 경계가 주제 클러스터를 가로지른다" 시그널이 재확인 — 타입 먼저가 아닌 주제·태그 먼저의 RAG 진입 구조가 더 적합하다는 근거 누적.
